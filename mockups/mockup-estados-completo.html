REPETIDA

Transiciones:
• +1 de faltante → queda como suelta (no asume que ya pegaste)
• Pegar al álbum de suelta → se vuelve pegada
• +1 de pegada → queda 1 pegada + 1 suelta extra (la extra puede apartarse para alguien)
• −1 resta primero de las sueltas; si no hay, baja la pegada
• Quitar todo vuelve a faltante

🔧 Cambios en el modelo de datos

Antes (hoy):

estado["COL-13"] = 3   // total en posesión, ambiguo

Después (con Suelta):

estado["COL-13"] = {
  pegada: 1,    // 0 o 1
  sueltas: 2    // copias en posesión sin pegar
}
// Total = pegada + sueltas
// Repetida cuando total > 1

Migración del estado actual al nuevo:

  • Si count = 0 → {pegada:0, sueltas:0}
  • Si count = 1 → {pegada:1, sueltas:0} (asumimos lo de hoy)
  • Si count ≥ 2 → {pegada:1, sueltas:count-1}

Una vez migrado, ustedes pueden ir corrigiendo "ah no, esa todavía no la pegué" mediante long-press → "Mover a suelta", a su ritmo, sin perder data.

🎯 Por qué este estado importa para los intercambios

Cuando hablamos de los 3 modelos de intercambio (Apartadas / Apartadas+persona / Tickets), la pregunta era: "¿de dónde sale la lámina que aparto/intercambio?"

→ De las sueltas, no de la pegada.
Una pegada está en el álbum, no se da. Solo las sueltas son apartables/intercambiables.

Por eso, este estado debería ir primero; los modelos de intercambio se construyen encima.
